###

<p align="left">Nosso projeto do tcc vem com objetivo de solucionar um problema existente no pais, a adocao de animais acontece por meios diretos entre ongs, redes sociais e conhecidos, com isso aumentando o risco para quem vai adotar, alem de nem sempre o adotante encontrar oq deseja, com isso pensamos e criamos um site onde junta todas as ongs em um unico lugar dando possibilidade de uma pessoa adotar um gato ou cachorro somente pesquisando em nosso site e assinando o formulario que seria enviado diretamente ao whatsapp da ong resposavel onde ela faria a validacao da adocao e avaliacao necessaria para saber se  o pet teria um bom local de moradia e convivio, nosso site aumigos carrega o peso e sonho de 4 pessoas que amam animais e sempre sentiram dificuldades e medo de adotar um animal diretamente nas redes sociais.</p>

<p align="left">metodologia do projeto Aumigos:<br><br>A equipe imaginou um projeto onde teria 5 entidades, ONG, Animal, membro da ONG, Adocao, Adotante, no inicio o uso do modelo MER/ERD foi de extrema ajuda para o Mateus dar inicio juntamente com a Leticia ao design do site, em seguida o mateus fez a definicao da linguagem  dentre php, python e javaScript o mateus escolheu javaScript por sua facilidade de uso no back e no front, em seguida o mateus juntamente com a agatha planejaram um banco de dados mysql que com o passar dos dia acabou atrassando demais o desenvolvimento do site e que levou o mateus a criar um novo banco usando o prototipo do primeiro mas trocando  o mysql pelo postgresql onde ele fez deploy no supabase e separou o proeto em duas partes, a parte da imagem e a parte de dados, no backend o mateus usou o motor node para criar o crud e se comunicar com APIs para validar o CNPJ  para saber se e uma ong de animais ou apenas uma organizca normal, com isso o mateus fez questao de validar essas informacoes por seguranca e apos a validacao cadasstrar no banco cada dado que vem do json entregue pela API BRASIL, usamos express para fazer essas conexoes, alem do bycript para criptografar as senhas antes de enviar ao banco, chegando n ofront tudo e bem simples mas seguro, cada usuario tem sua tela e nao da de forma alguma para outro usuario usar uma tela bloqueada pq ele sempre sera redirecionado e mesmo nao snedo ele nao consegue cadassstrar nada por nao ter um id correto dauqele banco de dados, o mateus usou o axios para pegar e enviar informacoes no frontend e fez deploy do backend e do frontend no render onde o projeto sempre e  iniciado ao entrar no link abaixo, talvez tendo perigo de o banco de dados parar de funcionar ou ser desligado no supabase, mas esse e nosso porjeto.</p>
